Bivariate Analysis — Finding What Drives Default

Le Thanh Binh · Data Scientist, Cake by VPBank · April 2026

EDACorrelationCredit Risk

Section 01What Is It & Why Do We Need It?

What is bivariate analysis?

Bivariate analysis examines pairs of variables — how one changes when the other changes, whether two features are correlated, and most importantly, how each feature relates to is_default. It is the bridge between "understanding each feature individually" and "building a model."

Why is it critical in banking?

Bivariate analysis in banking serves three distinct purposes simultaneously. First, it validates business logic: if credit_score and interest_rate are not negatively correlated — if the bank is charging higher rates to higher-credit borrowers — something is wrong with the data or the pricing model. Second, it identifies the most predictive features: a feature that shows strong separation between defaulters and non-defaulters is gold. Third, it detects multicollinearity: two features that are 0.95 correlated with each other carry nearly identical information, and including both in a linear model will destabilize the coefficients.

There are three types of bivariate relationships, each requiring different visualization:

Section 02Numerical × Numerical — Correlation Matrix

What is it?

A correlation matrix shows the pairwise Pearson correlation coefficient between every numerical feature — a value between –1 and +1 indicating how strongly two variables move together (positive) or in opposite directions (negative).

Why does it matter?

Two readings are important. First, correlation with the target: features strongly correlated with is_default are likely to be informative predictors. Second, correlation between features: two input features with correlation above 0.85 are nearly redundant — including both can destabilize linear models (multicollinearity) and may indicate they measure the same underlying construct.

Correlation Matrix — Numerical Features
Blue = positive · Red = negative · Intensity = strength · Diagonal is always 1.0

Key readings from this matrix: The –0.74 between credit_score and num_late_payments is expected and reassuring — high scorers have fewer late payments, confirming the bureau score is capturing real repayment behavior. The 0.55 between loan_amount and annual_income is also logical — higher earners borrow more. The –0.62 between credit_score and debt_to_income means high-DTI customers tend to have lower credit scores, which aligns with how credit bureaus calculate scores. None of these correlations are high enough to flag multicollinearity — the highest off-diagonal is 0.74.

When to worry about multicollinearity: Correlations above 0.85 between two input features create instability in linear and logistic regression models. Tree-based models (XGBoost, Random Forest) are more robust to it — but even there, near-duplicate features split training data redundantly and can reduce model interpretability. If credit_score and num_late_payments were at 0.93, you would drop the less business-interpretable one.

Section 03Numerical × Categorical — Income by Employment

What is it?

Comparing how a numerical feature is distributed across categories — typically using boxplots, which show the median, spread, and outliers for each group side by side.

Why does it matter?

This combination validates imputation decisions from Part 2 and reveals whether categorical features carry real discriminatory power for numerical ones. If annual_income distributions are identical across employment_type groups, segmented imputation adds no value. If they differ substantially, it validates the strategy of imputing missing income using employment-type-specific medians.

Annual Income Distribution by Employment Type
Boxplot · Confirms groups have different income levels — validates segment-specific imputation

The income distributions differ substantially across employment types. Freelancers have the lowest median income (52M VND/year) and the widest spread — highly irregular income is their defining characteristic. This confirms that global mean imputation for missing income would be wrong. Freelancer records with missing income should be imputed using the freelance median, not the salaried median.

Section 04Categorical × Target — Default Rates

What is it?

Computing and visualizing the default rate (percentage of loans that defaulted) for each category of a feature. This is the most direct bivariate look at which categorical features actually predict is_default.

Why does it matter?

This chart answers the central question of credit risk EDA: which features separate defaulters from non-defaulters? A feature where default rate is identical across all categories is a useless predictor. A feature where default rate varies by 10x across categories is extremely powerful.

Default Rate by Loan Grade (A → G)
Validates grading system monotonicity — each grade must default more than the previous

Monotonicity as a validation test: Default rates must increase monotonically from Grade A to Grade G. If they don't — if Grade D defaults less than Grade C — the grading model is broken. Never use a grading feature in a credit scoring model before running this check. A non-monotonic grade feature carries misleading signal that will confuse the model and produce nonsensical feature importances.

Section 05Weight of Evidence — The Bivariate Standard in Banking

What is it?

Weight of Evidence (WoE) is the banking industry's standard bivariate analysis technique for credit scoring. For each bin of a feature, WoE measures how much more (or less) likely defaulters are to fall in that bin compared to non-defaulters. A positive WoE means the bin has a higher proportion of defaulters; a negative WoE means it has fewer.

Why does it matter?

WoE analysis serves two purposes in EDA: it quantifies the predictive power of each feature (via the Information Value metric), and it reveals the shape of the relationship between a feature and default risk — which informs how the feature should be binned or transformed before modeling. Regulatory bodies also understand WoE analysis well, making it a natural language for model documentation.

WoE Pattern — Credit Score Bins
Negative WoE = bin has fewer defaulters than average · Positive = more defaulters than average

This WoE chart tells a clean, monotonic story: as credit score increases, WoE decreases from strongly positive (low-score borrowers are disproportionately defaulters) to strongly negative (high-score borrowers are much less likely to default). This monotonic pattern is both a validation that the feature is well-behaved and a signal that it is highly predictive of default risk.
